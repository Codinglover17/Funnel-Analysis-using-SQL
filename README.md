# 📊 A/B Testing Analysis using SQL

## 🧠 Project Overview

This project demonstrates how to simulate and analyze an A/B test using SQL.
Customers are divided into two groups — Control and Test — and their performance is compared using key business metrics.

---

## 🎯 Objective

To evaluate which group performs better by analyzing:

* Revenue
* Conversion Rate
* Average Revenue Per User (ARPU)

---

## ⚙️ Methodology

Since real experiment data is not available, we simulate an A/B test:

* Customers are split based on `customer_id`

  * Even IDs → **Control Group**
  * Odd IDs → **Test Group**

This deterministic approach ensures consistent and reproducible grouping.

---

## 📂 Dataset Used

The analysis uses two tables:

### 1. `sales.customers`

* `customerid`

### 2. `sales.orders`

* `customerid`
* `sales`

---

## 📈 Key Metrics Calculated

### 🔹 1. Total Revenue

Measures total sales generated by each group.

### 🔹 2. Conversion Rate

Percentage of users who made a purchase.

### 🔹 3. Average Revenue Per User (ARPU)

Average revenue generated per customer.

---

## 🔍 SQL Analysis Steps

1. Create Control and Test groups
2. Compare revenue between groups
3. Calculate conversion rate
4. Compute ARPU

All queries are available in:
👉 `ab_testing_analysis.sql`

---

## 💡 Key Insights

* Helps identify which variation (Control vs Test) performs better
* Provides data-driven decision support
* Useful for product feature evaluation and optimization

---

## 🛠 Tools Used

* SQL Server (SSMS)

---

## 🚀 How to Run

1. Open SQL Server Management Studio (SSMS)
2. Load the dataset (`sales.customers` and `sales.orders`)
3. Run the queries from `ab_testing_analysis.sql`
